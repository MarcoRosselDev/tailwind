# Que es la accesibilidad ?

La accesibilidad implica crear productos y servicios que todos pueden usar.
En el contexto del desarrollo web, es hacer sitios web que todos pueden
entender e interactuar, incluyendo personas con dificultades visuales, auditivas,
motoras y cognitivas.

Algunos ejemplos de descapasidades que pueden impactar la experiencia a los usuarios
online incluyen:

- cegera (blindness)
- poca vision (Low vision)
- Daltonismo (Color blindness)
- Sordera (deafness)
- Dificultad para usar el teclado, microfono o pantalla tactil (Difficulty using keyboards, mice, or touchscreens)
- Trastornos de atencion (Attention disorders)
- Problemas de memoria (Memory issues)
- Dificultad para hablar o entender el lenguaje hablado (Difficulty speaking or understanding spoken language)
- Sensibilidad a luces intermitentes (Sensitivity to flashing lights)

Hay solo unas pocas de muchas condiciones que pueden impactar a los usuarios
al rededor del mundo.

Para ayudarte a crear sitios web accessibles, El consorcio WWW o World Wide Web,
conosido como W3C, desarrollo un conjunto de estandares internacionales que puedes
seguir para hacer tu sitio web mas accesible y facil de usar para la gente
con discapasidades.

Estos estandares son conoosidos como "Web content accessibility guidelines"
(WCAG) o guia de accesisibilidad en contenido web.

Esta guia fue diseñada con cuatro centros principales en mente,
conosidos como POUR.

*P  (perceivable = perseptible)
	Los usuarios pueden persibir la informacion que estas presentando.
	Por ejemplo, puedes probeer texto alternativo para una imagen,
	por lo que los usuarios  que accedan a tu sitio web mediante una pantalla
	puedan leer y entenderlos.
*O 	(operable)
	Los usuarios deberan ser capaces de interactuar con la interfas de usuario.
	Por ejemplo, puedes asegurarte de que todas las funciolalidades son 
	accesibles a trabes del teclado tambien, no solo con el mouse.
*U 	(undestandable = entendible)
	Los usuarios deberan ser capaces de entender la informacion.
	Por ejemplo, puedes evitar sentencias complejas y usar lenguaje
	sensillo lo mas posible.
*R 	(Robust = robusto)
	Una amplia gama de navegadores y otras herramientas, incluidas las tecnologías 
	de asistencia, deben ser capaces de interpretar el contenido.

Usar HTML semantico ayuda mucho a mantener tu pagina web compatible con
diferentes navegadores y tecnologias de asistencia.

Si tu contenido no sigue ninguna de estos puntos principales, no todos
seran capaces de usar tu sitio web.

Para revisar si estas siguiendo esta guia correctamente,
puedes acceder a la Referencia Rapida de la WWW consortium.
Ahi podras encontrar una lista completa de tecnicas y criterios.

# Conclucion

La accesibilidad es esencial para los desarrolladores web.
Al desarrollar con inclusion en mente, te puedes hacegurar que todos pueden
usar y obtener tu contenido, promover igualidad y crear una mejor
experiencia de usuario para todos alrededor del mundo.
